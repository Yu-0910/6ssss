# サーバーは起動しているが接続できない問題

## 状況

ターミナルには以下のメッセージが表示されています：
```
✓ Ready in 14.8s
○ Local: http://localhost:3000
```

しかし、ブラウザで `http://localhost:3000` にアクセスしても接続できない。

## 確認手順

### 1. ポートが実際にリッスンしているか確認

```powershell
netstat -ano | findstr ":3000"
```

**期待される出力:**
```
TCP    0.0.0.0:3000           0.0.0.0:0              LISTENING       21168
```

### 2. ファイアウォールを確認

Windowsファイアウォールがポート3000をブロックしている可能性があります。

**確認方法:**
1. Windowsの設定 → プライバシーとセキュリティ → Windowsセキュリティ
2. ファイアウォールとネットワーク保護
3. 詳細設定
4. 受信の規則で「Node.js」または「ポート3000」を確認

### 3. ブラウザのキャッシュをクリア

- **ハードリロード**: `Ctrl + Shift + R`
- **シークレットモードで確認**: `Ctrl + Shift + N` (Chrome) または `Ctrl + Shift + P` (Firefox)

### 4. 別のブラウザで確認

- Chrome、Firefox、Edgeなど、別のブラウザで試す

### 5. プロキシ設定を確認

ブラウザのプロキシ設定が有効になっている場合、localhostへの接続がブロックされる可能性があります。

**Chromeの場合:**
1. 設定 → システム → コンピューターのプロキシ設定を開く
2. プロキシサーバーを使用しないで確認

## 解決方法

### 方法1: 別のポートを使用

```powershell
# サーバーを停止
Get-Process -Name node | Stop-Process -Force

# ポート3001で起動
npm run dev -- -p 3001
```

その後、`http://localhost:3001` にアクセス

### 方法2: 127.0.0.1を使用

`http://localhost:3000` ではなく、`http://127.0.0.1:3000` にアクセス

### 方法3: ネットワークアドレスを使用

ターミナルに表示されている Network アドレス（例: `http://10.68.109.186:3000`）にアクセス

### 方法4: ファイアウォールの例外を追加

1. Windowsの設定 → プライバシーとセキュリティ → Windowsセキュリティ
2. ファイアウォールとネットワーク保護 → 詳細設定
3. 受信の規則 → 新しい規則
4. ポート → TCP → 特定のローカルポート: 3000
5. 接続を許可する → すべてに適用

## デバッグ情報

サーバーが起動している場合、以下の情報を確認してください：

1. **ターミナルの完全なログ**
   - エラーメッセージがないか
   - コンパイルエラーがないか

2. **ブラウザの開発者ツール**
   - F12で開発者ツールを開く
   - Consoleタブでエラーを確認
   - Networkタブでリクエストの状態を確認

3. **Windowsイベントログ**
   - イベントビューアーでエラーを確認
