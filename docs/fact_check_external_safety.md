# 外部データソースとのファクトチェック - 安全性の確保

## 問題点

NPB公式サイトから選手リストを取得してCSVと比較する際、以下の問題が発生する可能性があります：

### 1. 取得できなかった選手の誤判定

**問題**: NPB公式サイトから取得できなかった選手を「存在しない」と誤って判断してしまう

**例**:
- 選手Aは実際に存在する
- しかし、NPB公式サイトから取得できなかった
- 比較結果では「CSVにのみ存在」と表示される
- 実際には「取得できなかった」だけの可能性がある

### 2. 取得失敗の見落とし

**問題**: 取得に失敗した年度・リーグを認識していないと、誤った結論を導いてしまう

**例**:
- 2025年PLのデータを取得しようとした
- しかし、NPB公式サイトから取得できなかった（404エラーなど）
- 取得できなかったことを記録していない
- 「CSVに不足している選手は0人」と誤って判断してしまう

## 解決策

### 1. 取得できなかった選手を記録

スクリプトは以下の情報を記録します：

- **取得できた選手**: NPB公式サイトから正常に取得できた選手
- **取得できなかった選手**: CSVには存在するが、NPB公式サイトから取得できなかった選手
- **取得失敗した年度・リーグ**: データ取得に失敗した年度・リーグのリスト

### 2. 比較結果の分類

比較結果は以下のように分類されます：

1. **CSVに不足（取得できたがCSVにない）**
   - NPB公式サイトから取得できた選手のうち、CSVに存在しない選手
   - **これは実際に不足している可能性が高い**

2. **CSVにあるが取得できなかった**
   - CSVには存在するが、NPB公式サイトから取得できなかった選手
   - **取得方法を確認する必要がある**

3. **CSVにのみ存在**
   - CSVには存在するが、NPB公式サイトから取得できなかった選手
   - ただし、全player_idリストに存在しない場合

### 3. 警告の表示

スクリプトは以下の警告を表示します：

```
⚠️ 重要な注意事項
取得できなかった選手が X人存在します。
これらの選手は、NPB公式サイトから取得できませんでしたが、
CSVには存在するため、取得方法を確認する必要があります。

「CSVに不足」と表示された選手のうち、
実際には取得できなかっただけの可能性があります。
```

## 使用方法

### 基本的な使用方法

```bash
node scripts/fact_check_npb_all_years.mjs --year 2025 --league PL
```

### 全年度・全リーグをチェック

```bash
node scripts/fact_check_npb_all_years.mjs
```

## 出力ファイル

### 1. `missing_players_{YEAR}_{LEAGUE}_external.csv`
取得できたがCSVに存在しない選手のリスト

### 2. `failed_to_fetch_{YEAR}_{LEAGUE}_external.csv`
CSVにあるが取得できなかった選手のリスト

### 3. `fact_check_{YEAR}_{LEAGUE}_external.json`
比較結果の詳細（JSON形式）

## 注意事項

1. **取得失敗の確認**: 必ず「取得できなかった選手」のリストを確認してください
2. **取得率の確認**: 取得できた選手数とCSV内の選手数を比較してください
3. **取得方法の改善**: 取得できなかった選手が多い場合は、取得方法を見直してください

## 推奨される確認手順

1. **取得結果の確認**
   - 取得できた選手数が妥当か確認
   - 取得失敗した年度・リーグがないか確認

2. **比較結果の確認**
   - 「CSVに不足」の選手を確認
   - 「取得できなかった」選手を確認

3. **取得方法の改善**
   - 取得できなかった選手が多い場合、取得方法を見直す
   - URL構造が変更されている可能性を確認

4. **再実行**
   - 取得方法を改善した後、再度実行して確認
