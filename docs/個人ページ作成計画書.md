# 個人ページ作成計画書

[baseballdata.jp](https://baseballdata.jp/) の選手個人ページ（例: [小園海斗 打撃成績詳細](https://baseballdata.jp/playerB/1800072.html)）のようなデータサイトの個人ページを、**Yahoo野球 スポーツナビ**（[baseball.yahoo.co.jp/npb/](https://baseball.yahoo.co.jp/npb/)）の試合ページを基に作成するための計画書である。

**前提**: 元となるサイト（Yahoo野球）は決定済み。**利用規約はクリア済み**。**過去年度の集計は不要**（当該年度のみを対象とする）。個人ページの**枠（ルーティング・コンポーネント骨格）は既に存在**するが、デザインは未決定。いったん表示させることを優先し、**デザイン編集は計画遂行後に人間が行う**。**パイロットテスト**として、[2026年3月4日開催のオープン戦5試合](https://baseball.yahoo.co.jp/npb/schedule/?date=2026-03-02)をスクレイピングし、パイプラインの検証を行う。

---

## 1. 目的

- **目標**: baseballdata.jp と同様に、**選手ごとの詳細成績ページ**（打撃・投手）を提供するデータサイトを構築する。
- **データ源**: Yahoo野球の試合ページ（例: [試合トップ](https://baseball.yahoo.co.jp/npb/game/2021040036/top)、[テキスト速報](https://baseball.yahoo.co.jp/npb/game/2021040036/text)）から取得したデータを集約し、個人ページを生成する。
- **対象年度**: 当該年度のみ。過去年度の遡及集計は行わない。

#### 5.0 試合ID取得方法（確立済み）

| 項目 | 内容 |
|------|------|
| **対象URL** | `https://baseball.yahoo.co.jp/npb/schedule/?date=YYYY-MM-DD` |
| **パラメータ** | `date`: 取得したい日付。指定日を含む週の日程を表示する。 |
| **抽出ロジック** | 1. 日程ページのHTMLを取得。2. リンク（`href`）から正規表現 `/npb/game/(\d{10})/` にマッチする部分を抽出。3. 重複を除いた10桁の数字を試合IDとする。 |
| **試合リンク例** | `https://baseball.yahoo.co.jp/npb/game/2021040036/index`、`/npb/game/2021040036/top` 等。IDは `2021040036`。 |
| **取得対象日** | パイロット: 2026年3月4日 → `?date=2026-03-02` または `?date=2026-03-04` で取得可能なページを1回以上リクエストし、3月4日の試合IDを抽出。本番: 当該年度の全日程について、日付をループして各日の試合IDを取得。 |
| **補足** | 1ページに複数日分が含まれる場合がある。取得した試合IDと日付の対応が必要なら、HTML内の日付表示（例: 「3月4日（水）」）と試合リンクの親要素をパースして紐づける。 |

### Phase 2: 打席データの構造化・正規化

#### 入力・出力

| 項目 | 内容 |
|------|------|
| **入力** | `_data/yahoo_games_pilot/` 内の HTML（`{game_id}_top.html`, `{game_id}_text.html`, `{game_id}_score.html`） |
| **処理** | `scripts/parse_yahoo_game_pilot.py` でパース → `scripts/normalize_plate_appearances.py` で正規化 |
| **出力** | 試合メタ `games_meta.csv`、打席単位 `plate_appearances.csv`、正規化済み `plate_appearances_normalized.csv` |

#### パース処理（`parse_yahoo_game_pilot.py`）

| ソース | 抽出内容 |
|--------|----------|
| **top** | game_id, date, stadium, game_time（試合開始時刻）, home_team, away_team, home_score, away_score（タイトル・メタから） |
| **text** | 各 `bb-liveText__item` から: game_id, inning, top_bottom（表/裏）, batter_id, **pitcher_id**（先発＋投手交代を追跡）, bat_order, base_state（無死走者なし等）, result_raw。試合前情報から先発投手、投手交代で継投を取得。 |
| **score** | 一球速報の球速・球種（Phase 2 では任意。Phase 3 で球種別集計時に活用） |

**ブロックD: セイバーメトリクス**

| 指標 | 算出式 | 参照 |
|------|--------|------|
| IsoP | SLG - AVG | `lib/ranking/leaders.ts` |
| IsoD | OBP - AVG | 同上 |
| BABIP | (H - HR) / (AB - SO - HR + SF) | 同上 |
| BB% | BB / PA × 100 | — |

| BB/K | BB / SO（SO>0 のとき） | — |

| SecA | (BB + (TB - H)) / AB | 同上 |
| TA | (TB + BB + HBP + SB) / (AB + BB + HBP + CS) | 同上 |

### 5.3 一球速報パイプライン（ブロックG: 球種・コース・25マス）

一球速報「詳しい投球内容」から**球種・球速・結果・コース（25マス）**を取得し、打席別表示・球種別成績として個人ページに反映するまでの一連の手順。同様の取得を他選手・他日付に拡張する際の手順書として記載する。

#### 5.3.1 スクレイピング

| 項目 | 内容 |
|------|------|
| **スクリプト** | `scripts/scrape_yahoo_pitch_details.py` |
| **対象URL** | `https://baseball.yahoo.co.jp/npb/game/{game_id}/score?index={index}` |
| **index形式** | `IIBT00`。II=イニング(01-09)、B=1表/2裏、T=打者番号(01-09)、00=固定。例: 1回表1番→`0110100` |
| **入力** | `plate_appearances_normalized.csv`（打席一覧。game_id, inning, top_bottom, bat_order から index を算出） |
| **出力** | `_data/yahoo_games_pilot/pitch_details.csv`（全打者）または `pitch_details_{選手名}.csv`（特定打者） |
| **オプション** | `--batter-id 1100082` で打者絞り込み（省略時=菊池のみ）。`--batter-id ''` で全員。`--date YYYY-MM-DD`、`--sleep 1.5` |
| **レート制限** | リクエスト間に sleep（デフォルト1.5秒）を入れる。打席数×sleep で所要時間を見積もる。 |

**パース対象HTML**:

- **投球テーブル**: `table.bb-splitsTable` の thead に「球種」「球速」「結果」を含む行。tbody の各行から: 投球番号、球種(`bb-splitsTable__data--ballType`)、球速、結果を抽出。
- **25マスゾーン**: `td.bb-splitsTable__allocationChartBg` 内の `div.bb-allocationChart` の `span.bb-icon__ballCircle`。各 span の `style="top:Xpx; left:Ypx;"` から座標を取得し、5×5グリッド（zone_row, zone_col, zone_id 1-25）に変換。視点は投手目線（投手がマウンドから見る視点）。
- **投手・打者ID**: `table#gm_rslt` 内の `/npb/player/{id}/top` リンクから取得。

**CSV出力列**: game_id, inning, top_bottom, bat_order, pitcher_id, batter_id, pitch_no, pitch_type, speed_kmh, result, zone_top_px, zone_left_px, zone_row, zone_col, zone_id

**実行例**:
```bash
python scripts/scrape_yahoo_pitch_details.py --date 2026-03-04 --batter-id 1100082
python scripts/scrape_yahoo_pitch_details.py --date 2026-03-04 --batter-id "" --out pitch_details_all.csv
```

#### 5.3.2 集計（球種別成績）

| 項目 | 内容 |
|------|------|
| **ロジック** | `lib/pitchDetailsPilot.ts` の `aggregateByPitchType()` |
| **入力** | `loadPitchDetails()` で読み込んだ打席別投球リスト |
| **出力** | `PitchTypeStats[]`: pitch_type, pitches, pct, avg_speed, balls, strikes, swing_miss, taken, foul, ab, h, avg |
| **打数・安打の算出** | 打席の**最終球**（決着球）の球種に打数・安打をカウント。`isSettlementResult()` で決着判定（左飛・ゴロ・三振・安打等）。`isHit()` で安打判定。 |
| **ボール・ストライーク** | 結果文字列の正規表現で分類。`/^ボール/`→ボール、`/^見逃し/`→見逃し、`/^空振り/`→空振り、`/^ファウル/`→ファウル。 |

#### 5.3.6 他選手・他日付への拡張

1. `plate_appearances_normalized.csv` に当該日の打席が含まれていること確認。
2. `scrape_yahoo_pitch_details.py` を `--batter-id {yahoo_id}` または `--batter-id ''` で実行し、対象打席の投球詳細を取得。
3. 出力CSVを `pitch_details_{選手id}.csv` 等に保存。`lib/pitchDetailsPilot.ts` の `loadPitchDetails()` が読み込むパスを、打者IDに応じて切り替えるか、単一CSVに全打者を蓄積する形に変更。
4. `getYahooIdForPilot()` に新選手のマッピングを追加するか、パイロット対象判定を緩和。
