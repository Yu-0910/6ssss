# Phase 1〜4 評価（2025年ランキングページ改善計画）

計画書 `docs/2025_ranking_page_improvement_plan.md` に基づき、Phase 1〜4 の実施内容と Phase 4 までの総合評価をまとめる。

---

## 1. 計画の達成条件（再掲）

1. **規定到達者CSV／全員版CSVの使い分け** … 指標ごとに2024年以前と同じ適用（規定必須＝規定用、カウント系＝全員版）。
2. **英字名前の表示** … ランキング行に `romanName` が表示される。
3. **2025年新規入団選手の反映** … 規定・非規定どちらのランキングにも漏れなく載る。

---

## 2. Phase 別評価

### Phase 1: 規定用CSV・全員版CSVの適用ルールの明確化と検証

| 項目 | 評価 | 内容 |
|------|------|------|
| 規定／全員の適用 | ✅ 達成 | 規定必須指標は `*_qualifying.csv` から `{metric}.json`、カウント系は全員版から `{metric}_all.json`。Client は `shouldRequireQualifyingPA` で取得先を切り替え。 |
| 2025年規定ルール | ✅ 達成 | `qualifying_rules.py` および `qualifyingPA.ts` の TEAM_GAMES_DATA に 2025（CL/PL 各143試合）を明示。minPA 443 で統一。 |
| 試合数マップ | ✅ 達成 | `config/games_per_team_by_season.json` を整備。ビルド時のフォールバック解消。 |
| 後方互換 | ✅ 達成 | Client 側の規定フィルタ（PA >= minPA）を維持。規定用CSV由来JSONでは全行通過（no-op）。 |

**結論**: Phase 1 は計画どおり完了。規定／全員の切り替えと2024年以前との一貫性が確保されている。

### Phase 3: 英字名前（romanName）の充足

| 項目 | 評価 | 内容 |
|------|------|------|
| 既存選手の流用 | ✅ 達成 | 2024年以前の from_master から英字名を収集し、2025年全員版の空欄行に流用（`phase3_fill_roman_names_2025.py` Step 1）。 |

**結論**: Phase 3 は完了。新入団選手を含め英字名が充足され、報告書と from_master の連携が取れている。

### Phase 4: ランキング再ビルドと表示の一貫性確認

| 項目 | 評価 | 内容 |
|------|------|------|
| 報告書→from_master 反映 | ✅ 達成 | `apply_report_en_to_from_master_2025.py` 実行により、CL/PL 各48行を報告書の英字名で更新。 |
| 規定用CSV再生成 | ✅ 達成 | `create_qualifying_csv_2025.py` で CL 18名・PL 22名の規定到達者CSVを生成。 |
| ランキングJSON再ビルド | ✅ 達成 | `build_rankings_2025_PL_full.py --year 2025 --league CL/PL` で2025年CL・PLの全指標JSONを再ビルド。 |
| 規定必須指標 | ✅ 達成 | `OPS.json` 等は規定到達者のみで、`romanName` が全行に含まれる（例: 柳町達 → T.Yanagimachi、佐藤輝明 → T.Satou）。 |

**結論**: Phase 4 は完了。Phase 1〜3 の変更が一連のビルドで反映され、規定／全員の切り替えと英字名・新入団選手の表示が一貫している。

## 4. 想定ネガティブポイントへの対応状況（Phase 4）

| ネガティブポイント | 対応状況 |
|--------------------|----------|
| 再ビルド後のキャッシュで古いJSONが返る | 開発時はハードリロードで確認。本番は既存の revalidate 等の設定に依存。必要に応じて Phase 5 で手順に明記。 |
| 非規定指標で100件に切られ新入団が下位だと出ない | 現状仕様として許容。計画書どおり Phase 5 以降で「表示件数」や「新入団のみ」の検討が可能。 |
| 規定用と全員版の選手数の差による混乱 | ヘルプやページ内説明の追記は Phase 5 のドキュメント整備で対応可能。 |

---

## 5. 残課題・推奨（Phase 5 以降）

1. **運用手順の明文化（Phase 5）**  
   報告書更新 → マージ → apply_report_en → 規定用CSV作成 → ランキング再ビルドの順を、実行コマンド例付きで1本の手順書にまとめる。

2. **キャッシュ・デプロイ**  
   本番で古いJSONが返る場合のキャッシュ無効化手順を、運用手順に含めるか検討。

3. **表示件数・新入団フラグ**  
   非規定指標の100件制限や「新入団のみ表示」は、必要に応じて Phase 5 以降で検討。
