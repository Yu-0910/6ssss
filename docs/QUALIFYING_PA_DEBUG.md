# 規定打席フィルタ実装が反映されない原因分析

## 現状確認

### 1. ファイルの状態
- `安打.json` と `安打_all.json` のハッシュが異なる
  - これは正規化処理が実行されていない、または失敗していることを示す

### 2. 実装済みのコード
- ✅ `normalize_metric_json_for_ui()` 関数は実装済み
- ✅ `validate_metric_json_mapping()` 関数は実装済み
- ✅ `main()` 関数で正規化処理が呼び出されている

## 考えられる原因

### 原因1: スクリプトが実行されていない
- 既存のJSONファイルが古いまま
- 正規化処理が実行されていない

**確認方法**: スクリプトを実行してログを確認

### 原因2: 指標名の判定が失敗している
- Record.csvから抽出される指標名と `METRICS_REQUIRE_QUALIFYING_PA_BY_NAME` の定義が一致していない可能性
- 例: Record.csvには "BABIP" だが、定義には "BABIP↓" が含まれている

**修正済み**: `METRICS_REQUIRE_QUALIFYING_PA_BY_NAME` から "BABIP↓" と "BB-K" を削除

### 原因3: ファイルパスの問題
- `output_dir` が正しく設定されていない
- ファイルが見つからない

**確認方法**: デバッグログでファイルパスを確認

### 原因4: Windowsのファイルロック
- ファイルが開かれているため上書きできない
- `tmp_path.replace(json_path)` が失敗している

**修正済み**: 既存ファイルを削除してからリネームする処理を追加

### 原因5: エラーが発生しているが気づいていない
- 正規化処理でエラーが発生しているが、ログが出力されていない
- 例外がキャッチされているが、処理が続行されている

**修正済み**: デバッグログとエラートレースバックを追加

## 次のステップ

1. **スクリプトを実行してログを確認**
   ```bash
   python scripts/build_rankings_2025_PL_full.py --year 2025 --league PL
   ```

2. **デバッグログを確認**
   - `[QUALIFY-MAP] デバッグ: 抽出された指標数=...`
   - `[QUALIFY-MAP] デバッグ: metric=... is_a_group=... is_b_group=...`
   - `[QUALIFY-MAP] B: overwrite ...`

3. **エラーログを確認**
   - `❌ 正規化エラー: ...`
   - トレースバック情報

4. **既存JSONファイルの状態を確認**
   - `安打.json` と `安打_all.json` のハッシュが一致するか確認
   - 一致していれば正規化済み、一致していなければ未実行

## 修正内容

1. ✅ `METRICS_REQUIRE_QUALIFYING_PA_BY_NAME` を Record.csv の実際の指標名に合わせて修正
2. ✅ デバッグログを追加（指標名の判定結果を出力）
3. ✅ Windowsのファイルロック対策を強化（既存ファイルを削除してからリネーム）
4. ✅ エラートレースバックを追加
