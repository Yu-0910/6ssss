# 記事の更新メカニズム - 現在の設定

## 📋 現在の設定のまとめ

### 記事の更新タイミング

**結論**: 記事は**自動更新されません**。以下のタイミングでのみ更新されます：

1. **ユーザーがページを開いた時**（初回読み込み時）
2. **キャッシュが1時間経過した後、ユーザーがページを開いた時**

---

## 🔍 詳細な動作

### 1. ページを開いた時（初回読み込み）

```
ユーザーがページを開く
  ↓
フロントエンド（ArticlesListClient）が `/api/articles` を呼び出す
  ↓
バックエンド（app/api/articles/route.ts）が処理を開始
  ↓
【キャッシュチェック】
  - キャッシュが1時間以内 → キャッシュされた記事を返す（外部API呼び出しなし）
  - キャッシュが1時間経過 or キャッシュなし → 外部APIから記事を取得
    ↓
    ・日刊スポーツのRSSフィードを取得（2-3フィード）
    ・auone.jpのHTMLを取得・パース
    ・各auone記事のOGP画像を取得（最大25件）
    ↓
    記事を取得・処理して返す
  ↓
フロントエンドが記事を表示
```

### 2. キャッシュの動作

**キャッシュ期間**: **1時間**（3600000ミリ秒）

**キャッシュの条件**:
- 本番環境（`NODE_ENV === 'production'`）
- デバッグモードではない（`debugMode !== true`）
- キャッシュクリアパラメータがない（`clearCache !== '1'`）
- キャッシュが1時間以内

**例**:
```
10:00 - ユーザーAがページを開く → 記事を取得（外部API呼び出し）
10:30 - ユーザーBがページを開く → キャッシュされた記事を返す（外部API呼び出しなし）
11:00 - ユーザーCがページを開く → キャッシュが1時間経過 → 記事を取得（外部API呼び出し）
11:30 - ユーザーDがページを開く → キャッシュされた記事を返す（外部API呼び出しなし）
```

---

## ⏰ 更新頻度のまとめ

| 状況 | 更新タイミング | 外部API呼び出し |
|------|--------------|----------------|
| ページを開いた時（キャッシュ有効） | なし（キャッシュを返す） | なし |
| ページを開いた時（キャッシュ無効） | あり（記事を取得） | あり（最大30回） |
| ページを開いた後 | なし（自動更新なし） | なし |
| 定期的な自動更新 | なし | なし |

---

## 🚫 現在実装されていない機能

### 1. 自動更新（定期的な更新）

**現状**: 実装されていません

**理由**:
- フロントエンドに `setInterval` や `setTimeout` が使用されていない
- バックグラウンドでの定期更新機能がない

**例**:
```
❌ 5分ごとに自動更新
❌ ページを開いている間、定期的に更新
❌ バックグラウンドで記事を取得
```

### 2. 手動更新ボタン

**現状**: 実装されていません

**理由**:
- フロントエンドに更新ボタンがない
- 更新ボタンを押した時の処理が実装されていない

## 🔧 設定ファイル

### `config/rss_feeds.json`

```json
{
  "settings": {
    "maxArticles": 50,
    "cacheDuration": 3600000,  // 1時間（ミリ秒）
    "timeout": 10000
  }
}
```

**`cacheDuration`**: キャッシュの有効期間（ミリ秒）
- `3600000` = 1時間
- `1800000` = 30分
- `600000` = 10分

### `app/api/articles/route.ts`

```typescript
export const revalidate = 0  // Next.jsのキャッシュを無効化（常に動的）
```

**`revalidate = 0`**: Next.jsのビルド時キャッシュを無効化
- 常に動的に記事を取得する
- メモリキャッシュ（1時間）は別途実装されている

## 💡 よくある質問

### Q1: 記事は何分ごとに更新されますか？

**A**: 自動更新はありません。ユーザーがページを開いた時のみ更新されます。

**ただし**:
- キャッシュが1時間以内の場合、キャッシュされた記事を返すため、外部API呼び出しは発生しません
- キャッシュが1時間経過した後、ユーザーがページを開くと、記事が更新されます

**実質的な更新頻度**:
- 最低でも1時間に1回（キャッシュが無効になった時）
- 実際には、ユーザーがページを開く頻度に依存

---

### Q2: ページを開いている間、記事は自動更新されますか？

**A**: いいえ、自動更新されません。

**理由**:
- フロントエンドに `setInterval` や `setTimeout` が使用されていない
- ページを開いた後は、記事の取得は行われません

**例**:
```
10:00 - ページを開く → 記事を表示
10:05 - ページを開いたまま → 記事は更新されない
10:10 - ページを開いたまま → 記事は更新されない
10:15 - ページをリロード → キャッシュが有効 → 同じ記事を表示
```

---

### Q3: 複数のユーザーが同時にページを開いた場合、何回外部APIが呼び出されますか？

**A**: キャッシュの状態によって異なります。

**例1: キャッシュが有効な場合**
```
10:00 - ユーザーAがページを開く → 記事を取得（外部API呼び出し: 1回）
10:05 - ユーザーBがページを開く → キャッシュを返す（外部API呼び出し: 0回）
10:10 - ユーザーCがページを開く → キャッシュを返す（外部API呼び出し: 0回）
```
**合計**: 1回

**例2: キャッシュが無効な場合**
```
10:00 - ユーザーAがページを開く → 記事を取得（外部API呼び出し: 1回）
11:00 - ユーザーBがページを開く → キャッシュが無効 → 記事を取得（外部API呼び出し: 1回）
11:05 - ユーザーCがページを開く → キャッシュを返す（外部API呼び出し: 0回）
```
**合計**: 2回

---

### Q4: キャッシュ期間を変更できますか？

**A**: はい、`config/rss_feeds.json` の `cacheDuration` を変更できます。

**例**:
```json
{
  "settings": {
    "cacheDuration": 1800000,  // 30分に変更
    ...
  }
}
```

**注意**:
- キャッシュ期間を短くすると、外部API呼び出しが増えます
- 外部サーバーへの負荷が増える可能性があります
- 推奨: 1時間（3600000ミリ秒）以上

---

### Q5: 手動で記事を更新する方法はありますか？

**A**: 現在は実装されていませんが、以下の方法で更新できます：

1. **URLパラメータを使用**（開発者向け）
   ```
   /api/articles?clearCache=1
   ```
   - キャッシュをクリアして、記事を再取得

2. **ページをリロード**
   - キャッシュが1時間経過していれば、記事が更新されます

3. **手動更新ボタンを実装**（将来的な機能）
   - フロントエンドに更新ボタンを追加
   - ボタンを押すと、キャッシュをクリアして記事を再取得

## 📝 まとめ

### 現在の設定

- **更新タイミング**: ユーザーがページを開いた時のみ
- **自動更新**: なし
- **キャッシュ期間**: 1時間
- **更新頻度**: 最低でも1時間に1回（キャッシュが無効になった時）

### 実質的な更新頻度

- **最低**: 1時間に1回（キャッシュが無効になった時）
- **実際**: ユーザーがページを開く頻度に依存
- **平均**: 複数のユーザーがいる場合、1時間に数回（最初のユーザーがキャッシュを更新）

### 推奨事項

1. **現在の設定で十分**: キャッシュ機構により、外部APIへの負荷は軽減されています
2. **手動更新ボタンの実装**: ユーザーが手動で更新できるようにする（リスク分析を参照）
3. **自動更新の実装**: 定期的に記事を更新する機能を追加する（将来的な機能）
