# H欠損とERA異常値 原因究明レポート

**調査日**: 2025年  
**調査スクリプト**: `scripts/investigate_h_era_issues.py`

---

## 1. H欠損の原因

### 1.1 症状

- **発生範囲**: 1950年〜2004年（2004年形式の選手ページ経由データ）
- **件数**: 年度・リーグあたり 1〜3件
- **合計**: 約55件（全152ファイル中）

### 1.2 該当行の特徴

| 特徴 | サンプル25件 |
|------|-------------|
| **IP（投球回）** | 全件が 空（None） |
| **G（登板）** | 全件が 1 または 2 |
| **BF（打者）** | 0〜5 程度の少ない値 |

**具体例**:
```
1950 CL 渡辺博之:   G=1  IP=空  ER=0  BF=4  BB=2  SO=2  ERA=4.0  H=空
1952 CL 大辻秀夫:   G=1  IP=空  ER=1  BF=4  BB=1  SO=2  ERA=3.0  H=空
1960 CL 金子哲夫:   G=1  IP=空  ER=0  BF=3  BB=3  SO=0  ERA=2.0  H=空
```

### 1.3 根本原因

1. **極少登板の投手**（1〜2試合のみ出場）では、NPB選手ページの投手成績表で**投球回が記録されない**（または「0」のみで安打列も未表示）
2. `scrape_2004_pitching_via_all_players.py` は**ヘッダー名ベース**で列をマッピングしている
3. 投球回が空の行では、**テーブル構造が通常行と異なる**（列数不足やセル結合など）可能性があり、「安打(H)」列を正しいインデックスで参照できない
4. 結果として、H列が空で取得される

### 1.4 結論

- **原因**: 1〜2登板のみの投手では、NPBサイト上で投球回・安打が未記録または別形式になっており、ヘッダー基準の列マッピングでHを正しく読めない
- **影響**: ごく少数（年間1〜2件程度）。ビッグデータ的には軽微

## 2. ERA異常値の原因

### 2.1 症状

- **発生範囲**:
  - **2004年以前**: 年度あたり 18〜34件。ERA 51〜190 の値
  - **2005年以降**: 年度あたり 1〜9件

### 2.2 該当行の特徴（2004年以前）

| 項目 | 値 |
|------|-----|
| **ER（自責点）** | 0 または 1（正常） |
| **IP（投球回）** | 100〜360 など（正常） |
| **ERA（CSV）** | 64〜184 など（異常に大きい） |

**具体例（藤本英雄 1950年）**:
- **NPB公式**: 防御率 **2.44**, 自責点 98, 失点 117
- **CSV**: ERA=**117.0**, ER=0（ERもずれている）

### 2.3 根本原因（2004年以前＝選手ページ経由）

NPB選手ページの**投手表**は年度により列構成が異なる。

| 年代 | ヘッダー | データ行 |
|------|----------|----------|
| **ホールド導入後**（1990年代〜） | 登板 勝利 敗北 セーブ **H** **HP** 完投 完封 … 失点 自責点 防御率 | H, HP 列あり |
| **ホールド導入前**（1950年代〜1980年代） | 同上（H, HP あり） | **H, HP 列なし**（完投・完封が直前） |

- ヘッダーには「H」「HP」が常にある
- 古い年度のデータ行には H, HP 列がなく、**実データは2列分左に詰まって並ぶ**
- 列インデックスで「防御率」を読むと、実際には**失点(R)**の値（例: 117）が読まれる
- **結果**: 失点が防御率として格納される → ERA 50超の異常値

### 2.4 根本原因（2005年以降＝球団別 idp1 / ロスター経由）

- **2005〜2024**: 球団別 idp1 のみ使用しており、列マップは修正済み。ERA異常は**ごく少数**（1〜9件/年度）
- **2025**: ロスター経由で取得。ERA異常が CL 8件・PL 9件とやや多い
- **想定**: 0回で失点したリリーフ投手で、NPB側が「無限大」を 99.99 や 999.00 などで表現している可能性

### 2.5 結論

| 年度範囲 | 原因 | 対策 |
|----------|------|------|
| **1950〜2004** | ホールド導入前のデータ行に H, HP がなく、2列ずれで失点がERAに格納される | 年代に応じて列オフセットを調整する |
| **2005年以降** | 0回で失点の「無限大ERA」表示が 50 超の数値になっている可能性 | 表示方式の確認。必要なら補正ロジックを検討 |

## 3. 推奨対応

### H欠損

1. **優先度: 低**（件数が少ないため）
2. 対応案: 投球回が空の行で、BF>0 かつ H が空の場合、別ソース（年度別一覧など）で補完する検討

### ERA異常値（2004年以前）

1. **`scrape_2004_pitching_via_all_players.py` の修正**
   - データ行のセル数で「ホールド導入前かどうか」を判定
   - ホールド導入前: **HOLD=0, HP=0** を設定（ホールド導入前は本来0であり、`Record_pitching.csv` の計算指標（ERA, WHIP, K%, BB%, K-BB%, WPCT）はいずれも HOLD・HP を使用しないため、派生指標への影響はない）
   - 完投〜防御率の列インデックスに **-2** のオフセットを適用
2. 修正後、**1950〜2004年**の**再スクレイピング**を実施
3. 計画書（`docs/投手ランキングページ作成のための報告書.md`）の Phase 1 に再スクレイピング計画を反映済み

### ERA異常値（2005年以降）

1. ERA>50 の行を数件サンプル抽出し、NPBサイトの表示と突き合わせ
2. 無限大ERAの扱い（例: 99.99 をそのまま格納するか、空にするか）を方針化
