## 2. ロールバック手順（要・人手実施）

問題発生時、本番を一時的に R2 参照から戻す、または R2 側を修復する手順。**実行判断と操作は人間が行う。**

### 前提

- **Vercel にランキングデータをデプロイして戻す**ことは行わない（データ量が大きく不可能なため）。
- ロールバックは次のいずれかで対応する。

---

### 案 A: R2 参照を一部だけにする（scope で絞る）

R2 に問題がある年度だけ 404 にしたい場合や、一時的に「参照する年度を絞りたい」場合。

1. **Vercel** にログイン → 対象 **Project** → **Settings** → **Environment Variables**
2. **RANKINGS_EXTERNALIZE_SCOPE** を追加または編集する。  
   - 例: `2024,2025` とすると、パスに 2024 または 2025 を含むリクエストだけ R2 に転送され、それ以外はプロキシ側の挙動（開発時はローカル、本番では 404 等）になる。  
   - **空または未設定** ＝ 全年度 R2 参照（現在の本番の形）。
3. **Save** 後、**Deployments** から本番を **Redeploy** する。
4. 再び全年度を R2 参照に戻す場合は、**RANKINGS_EXTERNALIZE_SCOPE** を削除するか空にして保存し、再度 Redeploy する。

※ scope の詳細は `lib/ranking/url.ts` および `app/data/rankings/[...path]/route.ts` を参照。

---

### 案 B: R2 の URL を切り替える（別バケット・修復後）

R2 のデータを修復した、または別バケット（バックアップ）の URL に切り替えたい場合。

1. **Vercel** → 対象 **Project** → **Settings** → **Environment Variables**
2. **RANKINGS_BASE_URL** の値を、修復済み R2 のベース URL、または別バケットの公開ベース URL に変更する（末尾に `/` は付けない）。
3. **Save** 後、本番を **Redeploy** する。
4. Phase 5 の動作確認チェックリストを再度実施し、表示と 200 を確認する。

---

### 案 C: 本番から R2 参照をやめる（ランキングを一時的に使えなくする）

**最終手段**。R2 も代替も使えない場合のみ。

1. **Vercel** → **Environment Variables** で、**Production** の **RANKINGS_BASE_URL** を**削除**する。
2. **Save** 後、本番を **Redeploy** する。
3. この状態では、プロキシは `RANKINGS_BASE_URL` が未設定のため 500 またはエラーレスポンスを返す想定。ランキングページはデータ取得に失敗する。
4. R2 または別 URL が使えるようになったら、**RANKINGS_BASE_URL** を再設定して Redeploy する。
