# 最新情報（記事フィード）実装計画書

## 概要

トップページの「最新情報」欄に、Yahoo!スポーツナビの「新着記事」のように、各スポーツ紙の最新記事を掲載する機能を実装する。

## 現状

- **現在の実装**: `app/page.tsx`で`dummyArticles`という固定データを使用
- **表示形式**: 画像 + タイトル + 日付 + 出典（スポーツ紙名）
- **タブ**: 「最新情報」タブ（`activeMainTab === 3`）で表示

## 目標

- 各スポーツ紙（スポーツニッポン、日刊スポーツ、スポーツ報知、デイリースポーツ、サンケイスポーツなど）の最新記事を自動取得
- リアルタイムまたは定期的に更新される記事リストを表示
- 既存のUIデザインを維持

## 実装方法の検討

### オプション1: RSSフィードの利用（推奨）

**メリット:**
- 多くのスポーツ紙がRSSフィードを提供している
- 利用規約に準拠しやすい
- 実装が比較的簡単

**デメリット:**
- RSSフィードが提供されていない場合がある
- 画像URLが含まれていない場合がある

**実装手順:**
1. 各スポーツ紙のRSSフィードURLを収集
2. サーバーサイド（API Route）でRSSフィードを取得・パース
3. 記事データを整形（タイトル、日付、出典、画像、リンク）
4. クライアントコンポーネントで表示

**必要な技術:**
- RSSパーサー（例: `rss-parser`）
- Next.js API Route
- キャッシュ戦略（更新頻度の制御）

### オプション2: スクレイピング

**メリット:**
- データソースの自由度が高い
- 画像URLも取得可能

**デメリット:**
- 利用規約違反のリスク
- サイト構造の変更に弱い
- レート制限の考慮が必要
- メンテナンスコストが高い

**実装手順:**
1. 各スポーツ紙のサイト構造を調査
2. スクレイピングスクリプトを作成
3. サーバーサイドで定期的に実行
4. データをキャッシュ（DBまたはJSONファイル）

### オプション3: 手動更新のCSV/JSONファイル

**メリット:**
- 実装が最も簡単
- 利用規約の問題がない
- 完全にコントロール可能

**デメリット:**
- 手動更新が必要
- リアルタイム性がない

**実装手順:**
1. CSVまたはJSONファイルで記事データを管理
2. サーバーサイドでファイルを読み込み
3. クライアントコンポーネントで表示

### オプション4: 外部APIの利用

**メリット:**
- 公式APIなら利用規約に準拠
- データの信頼性が高い

**デメリット:**
- 利用可能なAPIが限られる
- 有料APIの可能性
- APIキーの管理が必要

## 推奨実装方法

**段階的アプローチ:**

### Phase 1: 手動更新のJSONファイル（初期実装）
- 最も簡単で確実な方法
- 既存の`dummyArticles`をJSONファイルに移行
- 手動で記事を追加・更新

### Phase 2: RSSフィードの統合（本格実装）
- 各スポーツ紙のRSSフィードを調査
- RSSパーサーを実装
- 定期的な更新機能を追加

## データ構造

```typescript
type Article = {
  id: string | number
  title: string
  date: string // ISO 8601形式または "YYYY.MM.DD" 形式
  source: string // スポーツ紙名（例: "スポーツニッポン"）
  image: string // 画像URL
  link: string // 記事URL
  category?: string // カテゴリ（任意）
  publishedAt?: string // 公開日時（ISO 8601形式、任意）
}
```

## ファイル構成

```
app/
├── api/
│   └── articles/
│       └── route.ts          # 記事データ取得API
├── components/
│   └── ArticlesListClient.tsx # 記事リスト表示コンポーネント
_data/
└── articles/
    └── articles.json          # 記事データ（手動更新の場合）
```

## 実装手順（Phase 1: 手動更新）

1. **データファイルの作成**
   - `_data/articles/articles.json`を作成
   - 記事データをJSON形式で管理

2. **API Routeの作成**
   - `app/api/articles/route.ts`を作成
   - JSONファイルを読み込んで返す

3. **クライアントコンポーネントの作成**
   - `app/components/ArticlesListClient.tsx`を作成
   - APIからデータを取得して表示

4. **`app/page.tsx`の修正**
   - `dummyArticles`を削除
   - `ArticlesListClient`を使用

## 実装手順（Phase 2: RSSフィード）

1. **RSSフィードURLの収集**
   - 各スポーツ紙のRSSフィードURLを調査・収集

2. **RSSパーサーの実装**
   - `rss-parser`などのライブラリを使用
   - サーバーサイドでRSSフィードを取得・パース

3. **データ整形**
   - RSSデータを`Article`型に変換
   - 画像URLの取得（OGP画像など）

4. **キャッシュ戦略**
   - 一定時間（例: 1時間）キャッシュ
   - 定期的な更新機能

5. **エラーハンドリング**
   - RSSフィード取得失敗時のフォールバック
   - タイムアウト処理

## 技術スタック

- **RSSパーサー**: `rss-parser` (npm)
- **画像取得**: OGP画像の取得（`og-image`など）
- **キャッシュ**: Next.jsのキャッシュ機能またはRedis（本番環境）

## 注意事項

1. **利用規約の遵守**
   - 各スポーツ紙の利用規約を確認
   - RSSフィードの利用条件を遵守

2. **レート制限**
   - 過度なリクエストを避ける
   - 適切なキャッシュ戦略を実装

3. **エラーハンドリング**
   - RSSフィード取得失敗時の処理
   - フォールバックデータの準備

4. **パフォーマンス**
   - サーバーサイドでのデータ取得
   - 適切なキャッシュ戦略

## 次のステップ

1. **Phase 1の実装**（手動更新のJSONファイル）
   - 最も簡単で確実な方法から開始
   - 既存のUIを維持しつつ、データソースを変更

2. **RSSフィードの調査**
   - 各スポーツ紙のRSSフィードURLを収集
   - 利用規約を確認

3. **Phase 2の実装**（RSSフィード統合）
   - Phase 1が動作確認できたら、RSSフィードを統合
