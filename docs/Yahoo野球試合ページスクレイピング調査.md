# Yahoo野球 1試合ページ スクレイピング調査

[baseballdata.jp](https://baseballdata.jp/) のような野球データサイトを目指す前提で、[Yahoo野球 スポーツナビ](https://baseball.yahoo.co.jp/npb/) の1試合ページ（例: [2026年3月4日 オリックスvs広島](https://baseball.yahoo.co.jp/npb/game/2021040036/top)）をスクレイピングできるか調査した結果である。

---

## 1. スクレイピング可否の結論

| 観点 | 判定 | 備考 |
|------|------|------|
| **技術的可行性** | ◎ 可能 | HTMLは静的に取得可能。BeautifulSoup等でパースできる。テーブル・リンク構造が明確。 |
| **利用規約** | ⚠️ 要確認 | Yahoo! JAPANの利用規約・robots.txtの確認が必要。商用利用・二次配布の可否は要確認。 |
| **データの網羅性** | ◎ 豊富 | 試合概要・スコアボード・スタメン・責任投手・打順・打撃結果・投手交代・戦評・観客数・審判等が1試合あたり揃う。 |

---

## 2. 取得できる情報一覧（1試合あたり）

### 2.1 試合トップページ（`/game/{id}/top`）

| カテゴリ | 項目 | 取得可否 |
|----------|------|----------|
| **試合概要** | 日付・時刻・球場（京セラD大阪） | ◎ |
| | カード（オリックス vs 広島） | ◎ |
| | スコア（2-0）・試合結果 | ◎ |
| | 先攻・後攻 | ◎ |
| | 試合種別（オープン戦1回戦） | ◎ |
| **スコアボード** | イニング別得点（1〜9回・計） | ◎ |
| | 安打・失策 | ◎ |
| **責任投手** | 勝利・敗戦・セーブ投手（氏名・球団・勝敗S） | ◎ |
| **本塁打** | 本塁打を打った選手 | ◎ |
| **バッテリー** | 両軍の登板投手・捕手一覧（リンク付き） | ◎ |
| **戦評** | 試合のハイライトテキスト | ◎ |
| **スコアプレー** | 決勝点プレーの記述（例: 8回裏 杉澤のタイムリースリーベース） | ◎ |
| **エキサイティングプレーヤー** | 本日の成績・選考理由 | ◎ |
| **スターティング** | 両軍の打順・ポジション・選手名・打投・打率・調子 | ◎ |
| **審判** | 球審・塁審 | ◎ |
| **観客数・試合時間** | 15,198人・2時間25分 | ◎ |

### 2.2 関連サブページ

| URL | 内容 | 取得可否 |
|-----|------|----------|
| `/game/{id}/score` | 一球速報・試合中の打者・イニング別スコア | ◎ |
| `/game/{id}/score?index=0820100` | 一球速報（打席単位）。**「詳しい投球内容」に球速・球種・結果のテーブルあり** | ◎ |
| `/game/{id}/text` | 打席ごとのテキスト速報（打者・打球・結果・出塁・交代等） | ◎ |
| `/game/{id}/text#82` | 8回裏などイニング単位のアンカー | ◎ |

#### 一球速報「詳しい投球内容」で取得できる項目

各打席ごとに `?index=イニング表裏打者番号00` 形式でアクセスすると、以下のテーブルが取得可能。

| 列 | 例 |
|----|-----|
| 投球数 | 1, 2, 3, 4, 5 |
| 球種 | ストレート、カットボール |
| **球速** | **144km/h、145km/h、131km/h** |
| 結果 | ボール、ファウル、四球 |

球速帯別成績・球種別成績の集計が可能。

### 2.3 選手・球団ID

- 選手: `https://baseball.yahoo.co.jp/npb/player/{id}/top` 形式。IDはURLから取得可能。
- 球団: `https://baseball.yahoo.co.jp/npb/teams/{id}/index` 形式。
- 試合ID: `2021040036` のような形式（年度・試合番号等の規則は要分析）。

## 3. 発信できる情報（baseballdata.jp 風への応用）

[baseballdata.jp](https://baseballdata.jp/) は**シーズン通算・チーム・個人成績**中心だが、Yahoo試合ページからは**試合単位・打席単位**のデータが取れる。そのため、以下のような発信が可能。

### 3.1 試合単位で作れるコンテンツ

| 発信内容 | 説明 |
|----------|------|
| **試合結果サマリ** | 日付・カード・スコア・勝利・敗戦・セーブ投手・決勝プレー・観客数・試合時間を一覧化。 |
| **1試合あたりの投手成績** | 登板投手ごとのイニング・奪三振・与四球・失点等（テキスト速報から集計）。 |
| **1試合あたりの打撃成績** | 打席・安打・打点・四球等（同上）。 |
| **決勝点・殊勲打** | スコアプレーの記述から決勝打者・イニング・状況を抽出。 |

### 3.2 複数試合を集約して作れるコンテンツ

| 発信内容 | 説明 |
|----------|------|
| **日程・結果一覧** | 試合IDを収集し、日付・カード・スコア・責任投手を表形式で表示。 |
| **観客動員・試合時間** | baseballdata.jp の「観客動員＆試合時間」と同種の集計。 |
| **責任投手ランキング** | 勝利・敗戦・セーブを試合ごとに集計し、シーズン通算を算出。 |
| **決勝打ランキング** | 決勝打者を試合ごとに記録し、殊勲打ランキングを構築（baseballdata.jp の「殊勲打ランキング」に相当）。 |
| **得点圏・勝負所の打率** | テキスト速報から「○死○塁」「先制の場面」等の打席を抽出し、得点圏打率・UC打率風の指標を算出（baseballdata.jp の「得点圏被打率」「VDUCP(UC)」に近い発想）。 |

### 3.3 一球速報・テキスト速報を活用した高度な発信

| 発信内容 | 説明 |
|----------|------|
| **球速・球種別成績** | 一球速報「詳しい投球内容」から球速・球種・結果を取得し、球速帯別・球種別の打率・打数等を集計。 |
| **打席単位の成績DB** | 打者・投手・打席結果・打球方向・球種（一球速報＋テキスト記述）を構造化。 |
| **投手交代・継投の分析** | イニング・打者・投手交代の記録から継投パターン・起用傾向を分析。 |
| **球種・コースの記述分析** | 「外角高めのストレート」等の記述＋一球速報の球種から、球種・コース別成績を集計。 |
| **イニング別・得点差別の成績** | イニング・得点差を打席ごとに付与し、状況別成績を集計。 |

## 4. 実装上の留意点

| 項目 | 内容 |
|------|------|
| **試合IDの収集** | 日程ページ（`/npb/schedule/`）や他試合リンクから試合IDを収集する必要あり。 |
| **動的コンテンツ** | 一球速報（`/score`）は試合中に更新されるが、試合終了後は静的になっている想定。必要に応じてJavaScriptレンダリングやAPIの有無を確認。 |
| **レート制限** | 1試合あたり複数リクエスト（top, score, text）になるため、sleep・リトライを設定。 |
| **HTMLの変更** | Yahoo側のリニューアルで構造が変わる可能性。定期的な動作確認が望ましい。 |
| **データの正確性** | 公式発表と突き合わせて検証する運用を推奨。 |

---

## 5. まとめ

- **スクレイピング**：技術的には可能。HTMLがパースしやすく、1試合あたりの情報量も多い。
- **利用規約**：Yahoo! JAPANの規約・robots.txtの確認が前提。
- **発信の切り口**：試合結果・責任投手・決勝打・観客数・試合時間などの試合単位データを集約し、baseballdata.jp と同様の「観客動員＆試合時間」「殊勲打ランキング」や、テキスト速報由来の「得点圏・勝負所の打率」「打席単位DB」といった独自コンテンツを構築できる。
