### データの流れ

```
master_csv__import_1950_2024 (スクレイピング生データ)
    ↓
master_csv (現在使用中の生データ)
    ↓
compute_metrics_all_seasons.py (計算処理)
    ↓
master_csv_calculated (計算済みデータ)
```

すべての段階で、若松勉の1971-1985年のデータが存在しています。

### 考えられる原因（UIで表示されない場合）

もしUIで若松勉の1971-1985年の記録が表示されていない場合は、以下の原因が考えられます：

1. **ランキング生成スクリプトが特定の年度範囲のみを処理している**
   - `build_rankings_*.py` が特定の年度のみを対象としている可能性
   - 年度フィルタが設定されている可能性

2. **UIの表示ロジックが特定の年度範囲のみを対象としている**
   - フロントエンドのコードが特定の年度範囲のみを読み込んでいる可能性
   - 年度選択UIで1971-1985年が選択できない可能性

3. **データは存在するが、読み込み時にフィルタリングされている**
   - データ読み込み時に年度フィルタが適用されている可能性
   - 最小打席数などの条件で除外されている可能性

### 確認方法

1. **CSVファイルの直接確認**
   ```powershell
   # 1971年のデータを確認
   Select-String -Path "_data\master_csv_calculated\batting_1971_CL_from_master.csv" -Pattern "11613843"
   ```

2. **ランキング生成スクリプトの確認**
   - `scripts/build_rankings_*.py` の年度フィルタを確認
   - 処理対象年度の範囲を確認

3. **UIコードの確認**
   - フロントエンドの年度選択ロジックを確認
   - データ読み込み時のフィルタを確認

### 結論

**計算後に1971-1985年の記録が消えたということはありません。**

すべてのディレクトリで、若松勉の1971-1989年の記録が確認できました。データは正常に存在しており、計算処理によってデータが失われることはありませんでした。

もしUIで表示されていない場合は、ランキング生成スクリプトやUIの表示ロジックに原因がある可能性があります。
