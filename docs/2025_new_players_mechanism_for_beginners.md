# 2025年「新入団選手だけ」をランキングに追加する仕組み（初心者向け）

ランキングの「元データ」には**全選手の成績**が入ったCSVがあります。  
ここに**新入団選手の行が足りない**とき、「新入団選手**だけ**」を追加する仕組みを、初心者にも分かるように説明します。

---

## 用語の説明（ここだけ押さえておく）

| 用語 | 意味（やさしく言うと） |
|------|------------------------|
| **CSV** | 表形式のデータファイル（Excelで開ける）。1行＝1選手の成績。 |
| **全員版CSV** | その年・そのリーグの**出場した選手ぜんぶ**が1行ずつ入っている表。ランキングの「全選手」表示の元。 |
| **規定到達者** | 打席数などが「規定」を満たした選手だけ。率のランキング（打率・OPSなど）はこの人たちだけで争う。 |
| **報告書** | このプロジェクトでは「2025年に新しく入った選手の名前リスト」のCSVのこと。 |
| **バックアップ** | 昔の状態を取っておいたファイル。ここから「新入団選手の行だけ」を取り出して使うことがある。 |

---

## 何が問題で、何をしているか（大きな流れ）

- **問題**: 2025年の「全選手の表」は、別のところ（npb_batting）から持ってきたので、**新入団の何人かが最初から入っていない**ことがある。
- **やっていること**: 「新入団選手**だけ**」をリスト（報告書）で管理し、その人たちの行が表に無ければ**追加**する。成績は「バックアップに同じ選手の行があればそれ」、無ければ「名前とチームだけ入れた行」で追加する。

つまり、**全選手を入れ直すのではなく、「新入団と分かっている人だけ」を足している**のが「新入団選手に限って追加した」の中身です。

---

## 「新入団選手に限って」追加する仕組み（3つのパーツ）

### 1. 誰を「新入団」とするか → **報告書**

- **ファイル**: `_data/reports/2025_new_players_report.csv`
- **中身**: 「2025年に新しく入った選手」の名前・チーム・英字名などが1行ずつ並んだ表。
- **役割**: 「この表に名前がある人**だけ**を、全員版CSVに足す」という**リスト**。  
  → ここに載っていない人は、この仕組みでは追加しない（＝新入団に限っている）。

### 2. 追加する「成績」はどこから取るか → **バックアップ**

- **フォルダ**: `_data/master_csv_calculated/_backup_2025_from_master/`
- **中身**: 昔保存しておいた「2025年セ・リーグ用」「2025年パ・リーグ用」の成績表（CSV）。
- **役割**:  
  - 報告書に載っている選手の行が、このバックアップの表に**ある** → その行（成績付き）を今の全員版CSVに**コピーして追加**。  
  - バックアップに**ない**（例: チェイビス） → 成績は入れず、「名前・チーム・年・リーグ」だけの行を追加（数字は0のまま）。  

→ つまり「新入団の成績の一次ソース」は、**バックアップのCSV**。ここに無い人だけ、後から手で成績を入れるか、最小限の行で追加している。

### 3. 実際に「足す」処理をしているもの → **マージ用スクリプト**

「報告書を見て、全員版CSVにいない人だけを、バックアップから取るか最小行で足す」プログラムがいくつかあります。

| スクリプト | 何をするか（やさしく） |
|------------|------------------------|
| `merge_2025_new_players_from_report.py` | 報告書を開いて、全員版CSVにいない選手**だけ**を探し、バックアップにあればその行を、無ければ「名前だけ」の行を足す。 |
| `merge_2025_new_players_into_cl_from_master.py` | セ・リーグ用。最初は3人（ファビアン・キャベッジ・ボスラー）だけ、バックアップからセの全員版CSVに足していた。 |
| `merge_2025_new_players_and_build_rankings.py` | 上と同じように「報告書の選手だけ」を足したあと、ランキング用のJSONを自動で作り直す。 |
| `build_2025_full_from_master_and_rankings.py` | 2025年のデータを一通り更新する手順のなかで、「報告書の選手で足りない分を追加（バックアップにあれば成績も反映）」をやる。 |

どれも「**報告書に載っている新入団選手に限って**」追加しているのがポイントです。

## 図で見る流れ（初心者向け）

```
[報告書] 2025_new_players_report.csv
  「この人たちが新入団」のリスト
        ↓
  「全員版CSVに、この人いる？」とチェック
        ↓
  いない人だけ ↓
        ↓
[バックアップ] _backup_2025_from_master/*.csv
  「その人の行、ここにある？」 → あればその行をコピー
                               → なければ「名前・チームだけ」の行を作る
        ↓
[全員版CSV] batting_2025_*_from_master.csv に追加
        ↓
  ここからランキング用のJSONを作り直す → サイトに反映
```

## 質問2・4・8の答え（初心者向け）

- **質問2（新入団の成績の一次ソース）**  
  → 「新入団選手に限って追加した」ときの成績は、**バックアップのCSV**から取っている。バックアップに無い人だけ、後から手で入れるか「名前だけ」の行で追加している。

- **質問4（2025年の規定打席・試合数）**  
  → **同じで良い**。2024年までと同じルール・同じ計算のまま、2025年も扱う。

- **質問8（対象は2025年だけ？ 全年度？）**  
  → **2025年だけ**。昔の年度のランキングやデータは触らず、2025年セ・パのランキングだけを整える。

## 自分で「新入団を1人足したい」とき（初心者向けの手順イメージ）

1. 報告書（`2025_new_players_report.csv`）を開く。  
2. いちばん下に、新しい1行を足す。  
   - 選手名（日本語）・チーム・英字名 などを、他の行と同じ列に入れる。  
3. 保存したあと、`merge_2025_new_players_from_report.py` を実行する。  
4. 必要なら、規定用CSVの作り直し → ランキングJSONの作り直し、までやる（DATA_PATHS.md にコマンドが書いてある）。

「誰を新入団とするか」は報告書で管理し、「その人たちだけ」を全員版CSVに足す、という形になっています。
