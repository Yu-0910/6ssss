# Cursor運用ルール（固定・今後すべてこれに従う）

## 前提

- 本プロジェクトでは、実装・修正・検証はすべて Cursor 向け指示で行う
- 本ルールは今後の全タスクに適用される

## 【基本方針】

- UIは触らない
- データと検証のみを扱う
- 仕様書（docs/RANKING_PAGE_INFO.md）を唯一の正とする
- 「動いたからOK」は禁止。validateが通ることを成功条件とする

## 【出力形式】

- 実装・修正・検証は必ず Cursor 向けプロンプトとして実行
- 1タスク = 1プロンプト
- そのままコピー＆ペーストで使えるMarkdown形式のみ
- 余計な説明や改善提案は不要

## 【変更してよい範囲】

### OK:
- scripts/
- config/
- lib/ranking/
- public/data/

### NG（明示指示がない限り）:
- components/（UI）
- app/ のレイアウト・構造
- CSS / Tailwind / デザイン関連
- Record.csv の順番・意味

## 【禁止事項】

- UI構造や表示ロジックの変更
- データ条件によるUI分岐の追加
- 「ついでにリファクタ」「より良くする」系の変更
- 指標の並び順変更
- Record.csv を直接書き換える行為

## 【データに関する絶対ルール】

- 指標定義は Record.csv が唯一の正
- 表示名 → 内部キー変換は config/metric_map.json のみ
- Python / TypeScript でキー定義を重複させない
- JSON生成後は必ず validate を実行し、欠損があれば例外で停止

## 【失敗時の扱い】

- validateでエラーが出る → 正常（再発防止が機能している）
- 画面で初めて気づく状態 → 異常（修正対象）
- エラーを握り潰して先に進むことは禁止

## 【作業フロー】

1. Cursor向けプロンプトを貼る
2. Cursorが実装
3. 人間が実行
4. 結果（OK / エラーログ）をそのまま共有
5. 次のCursor向けプロンプトを作成

## 【最優先目標】

- 「以前と同じミス（キー不一致・指標欠け・UI崩壊）」を二度と起こさない
- 正しさは validate で保証する
