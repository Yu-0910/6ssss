# プレイヤープロフィール情報のスクレイピング可能性分析

## NPB公式サイトからスクレイピング可能な情報

### ✅ 直接取得可能
1. **生年月日**
   - NPB公式サイトの選手プロフィールページに記載
   - URL例: `https://npb.jp/bis/players/01705130.html`
   - 年齢は生年月日から計算可能

2. **出身地**
   - NPB公式サイトの選手プロフィールページに記載
   - 出身高校・出身地の情報が含まれる

3. **プロ入り（ドラフト情報）**
   - NPB公式サイトの選手プロフィールページに記載
   - ドラフト年、ドラフト順位（例：ドラフト2位）は取得可能
   - ⚠️ **注意**: 「全体14位」のような全体順位は記載されていない可能性が高い
     - ドラフト全体順位は、各球団のドラフト順位を集計して計算する必要がある

4. **経歴**
   - NPB公式サイトの選手プロフィールページに記載
   - 高校 → 大学 → 社会人 → プロの経歴情報が含まれる

## NPB公式サイトから直接取得できない情報

### ❌ スクレイピング不可（別データソースまたは計算が必要）

1. **生涯年俸**
   - NPB公式サイトには各年度の年俸情報が記載されていない
   - 各年度の年俸データを別途収集し、合計する必要がある
   - データソース候補：
     - スポーツ新聞の年俸特集記事
     - 各球団公式サイト（一部公開している場合）
     - 野球専門メディア

2. **チーム成績（日本一：5回、リーグ優勝：7回）**
   - これは選手個人の成績ではなく、所属チームの成績
   - NPB公式サイトにはチームの優勝歴はあるが、選手個人ページには表示されていない
   - データソース候補：
     - NPB公式サイトのチーム成績ページ
     - 選手の所属年とチームの優勝年を照合して計算

3. **FA取得（推定）**
   - NPB公式サイトには記載されていない
   - 計算で求める必要がある：
     - 入団年 + 9年（高校卒業後入団の場合）
     - 入団年 + 8年（大学卒業後入団の場合）
     - 入団年 + 7年（社会人経由の場合）
   - 実際のFA取得年は契約状況により変動する可能性がある

## 実装推奨事項

### スクレイピング実装可能な項目
```typescript
// NPB公式サイトから取得可能
interface ScrapableProfile {
  生年月日: string        // ✅ スクレイピング可能
  出身地: string          // ✅ スクレイピング可能
  プロ入り: string        // ✅ スクレイピング可能（ドラフト順位まで）
  経歴: string            // ✅ スクレイピング可能
}
```

### 別データソースまたは計算が必要な項目
```typescript
// 別途データ収集または計算が必要
interface CalculatedProfile {
  生涯年俸: string        // ❌ 別データソース必要
  チーム成績: string      // ❌ 別データソース必要（チーム成績ページから）
  FA取得年: string        // ❌ 計算で求める（入団年+年数）
}
```

### ドラフト全体順位について
- NPB公式サイトには「ドラフト2位」のような球団内順位は記載されている
- 「全体14位」は、全12球団のドラフト順位を集計して計算する必要がある
- 実装方法：
  1. 各球団のドラフト順位を取得
  2. ドラフト全体の順位を計算（例：1巡目1位=1位、1巡目2位=2位...2巡目1位=13位、2巡目2位=14位）
  3. または、ドラフト全体順位のデータベースを別途作成

## データ取得の優先順位

1. **最優先（スクレイピング実装）**
   - 生年月日
   - 出身地
   - プロ入り（ドラフト順位）
   - 経歴

2. **次段階（別データソース検討）**
   - 生涯年俸（年俸データベースの構築）
   - チーム成績（チーム成績ページからの取得）

3. **計算で求める**
   - FA取得年（入団年から計算）
