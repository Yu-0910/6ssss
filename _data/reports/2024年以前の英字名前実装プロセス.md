# 2024年以前の英字名前実装プロセス報告書

## 概要

2024年以前の選手データに対して、NPB公式サイトから英字名前（romanName）を収集し、成績CSVファイルに反映するまでの実装プロセスをまとめた報告書です。

## 実装段階

### STEP 1: player_idリストの準備

**目的**: 全選手の`player_id`を収集し、処理対象を明確化する

**処理内容**:
- 既存の成績CSVファイル（`_data/master_csv/`、`_data/master_csv_calculated/`）から`player_id`を抽出
- ユニークな`player_id`のリストを作成
- 出力: `output/master/all_player_ids.csv`

**必要なスクリプト**: （既存のCSVから抽出するスクリプト、または手動でリスト化）

---

### STEP 2: NPB公式サイトの構造調査

**目的**: NPB公式サイトの選手個人ページから英字名前を取得する方法を確立する

**処理内容**:
- `scripts/probe_player_page_name_fields.py`を使用して、NPB公式サイトの選手個人ページ（`https://npb.jp/bis/players/{player_id}.html`）のHTML構造を調査
- 以下の要素から英字名前を抽出可能であることを確認:
  - `id="pc_v_name"`の`<li>`要素内の括弧内の英字（外国人選手用）
  - `id="pc_v_kana"`の`<li>`要素内の括弧内の英字（日本人選手用）
  - `id="pc_v_kana"`の`<li>`要素内の直接の英字（外国人選手用、括弧なし）

**出力**: 調査結果のレポート

---

### STEP 3: 公式かな・英字の収集

**スクリプト**: `scripts/build_player_name_kana_and_official_roman.py`

**目的**: 全`player_id`について、NPB公式サイトから読み仮名（かな）と英字表記（roman_official）を収集する

**処理内容**:

1. **HTML取得**:
   - 各`player_id`に対して、複数のURL候補を生成:
     - `https://npb.jp/bis/players/{player_id}.html`
     - `https://npb.jp/bis/players/{player_id:08d}.html`（ゼロ埋め8桁）
     - `https://npb.jp/bis/players/{player_id:07d}.html`（ゼロ埋め7桁）
   - リトライ機能付きでHTMLを取得（最大5回リトライ）
   - レート制限を考慮（デフォルト1秒間隔）

2. **HTMLキャッシュ**:
   - 取得したHTMLを`output/html_cache/players/`に保存
   - 次回実行時にキャッシュから読み込み可能（`--use-cache`オプション）

3. **データ抽出**:
   - `find_japanese_name()`: 日本語名を抽出（`id="pc_v_name"`の`<li>`要素から）
   - `find_kana_name()`: 読み仮名を抽出（`id="pc_v_kana"`の`<li>`要素から、括弧前の部分）
   - `find_roman_name()`: 英字表記を抽出（括弧内の英字、または直接の英字）

4. **継続実行機能**:
   - `--resume`オプションで既存の結果CSVから処理済み`player_id`をスキップ
   - 中断されても続きから実行可能

**出力**: `output/master/player_id_name_kana_official.csv`
- 列: `player_id`, `name_ja`, `name_kana`, `roman_official`, `url_used`, `http_status`, `outcome`

**実行例**:
```bash
python scripts/build_player_name_kana_and_official_roman.py \
  --ids output/master/all_player_ids.csv \
  --out output/master/player_id_name_kana_official.csv \
  --rate 1.0 \
  --resume \
  --use-cache
```

**統計情報**:
- `name_kana`非空率: 取得できた読み仮名の割合
- `roman_official`非空率: 取得できた英字表記の割合
- `outcome`別件数: OK、NAME_JA_ONLY、NO_DATA、FAILEDなど

---

### STEP 4: かな→ローマ字変換 & 最終辞書作成

**スクリプト**: `scripts/build_player_id_to_roman_full.py`

**目的**: 読み仮名（かな）をヘボン式ローマ字に変換し、最終的な「イニシャル.苗字」形式の辞書を作成する

**処理内容**:

1. **入力データの統合**:
   - `player_id_name_kana_official.csv`を読み込み
   - 同一`player_id`が複数行ある場合、以下の優先順位で統合:
     1. `roman_official`が非空の行（最優先）
     2. `name_kana`が非空の行
     3. `name_ja`が非空の行

2. **英字名前の生成**:
   - **優先1**: `roman_official`が存在する場合
     - Title Caseに変換（例: "ROMAN MEJIAS" → "Roman Mejias"）
     - 外国人選手として処理: `to_initial_lastname(full_name, is_japanese=False)`
     - 例: "Roman Mejias" → "R.Mejias"（名前のイニシャル.苗字）
   
   - **優先2**: `name_kana`が存在する場合
     - ヘボン式ローマ字変換テーブル（`HEPBURN_TABLE`）を使用してかなをローマ字に変換
     - `convert_kana_to_romaji()`: 「・」区切りのかなを「苗字 名前」形式のローマ字に変換
     - 日本人選手として処理: `to_initial_lastname(full_name, is_japanese=True)`
     - 例: "さとう・てるあき" → "Sato Teruaki" → "T.Sato"（名前のイニシャル.苗字）

3. **ヘボン式ローマ字変換の詳細**:
   - ひらがな・カタカナをヘボン式ローマ字に変換
   - 長音記号（ー）の処理: 前の文字の母音を長音化
   - 促音（っ）の処理: 次の子音を重ねる
   - 拗音（きゃ、しゅなど）の処理: 2文字の組み合わせを優先的にチェック
   - Title Caseに変換（各単語の先頭を大文字、他を小文字）

4. **イニシャル.苗字形式への変換**:
   - `to_initial_lastname()`関数で「イニシャル.苗字」形式に変換
   - 日本人選手: 「苗字 名前」→「名前のイニシャル.苗字」
   - 外国人選手: 「名前 苗字」→「名前のイニシャル.苗字」

**出力**: `output/master/player_id_to_roman_full.csv`
- 列: `player_id`, `romanName`, `source`, `confidence`, `name_ja`, `name_kana`
- `source`: `NPB_OFFICIAL`（公式英字から）または`KANA_CONVERTED`（かなから変換）
- `confidence`: `HIGH`（信頼度が高い）または`LOW`（データ不足）

**実行例**:
```bash
python scripts/build_player_id_to_roman_full.py \
  --input output/master/player_id_name_kana_official.csv \
  --output output/master/player_id_to_roman_full.csv
```

**統計情報**:
- 総`player_id`数
- `romanName`が埋まった数と割合
- `source`別内訳（NPB_OFFICIAL、KANA_CONVERTED、MISSING）

---

### STEP 5: 成績CSVへの反映

**スクリプト**: `scripts/apply_roman_to_master_csvs.py`

**目的**: 最終辞書（`player_id_to_roman_full.csv`）の`romanName`を成績CSVファイルに反映する

**処理内容**:

1. **辞書の読み込み**:
   - `output/master/player_id_to_roman_full.csv`を読み込み
   - `player_id` → `romanName`のマッピングを作成

2. **成績CSVファイルの検索**:
   - `_data/master_csv/`と`_data/master_csv_calculated/`から
   - `batting_YYYY_(PL|CL)_from_master.csv`パターンのファイルを検索

3. **CSVファイルへの適用**:
   - 各CSVファイルを読み込み
   - `player_id`列を探す（`player_id`, `playerId`, `id`など）
   - `player_name_en`列を探す（存在しない場合は追加）
   - 各行について:
     - `player_id`が辞書に存在する場合、`romanName`を設定
     - 既に「イニシャル.苗字」形式の値がある場合はスキップ
     - フルネーム形式（例: "Nishikawa Manaya"）の場合は辞書の値で上書き

4. **文字コード処理**:
   - 入力ファイルの文字コードを自動検出（`utf-8-sig`, `utf-8`, `shift_jis`, `cp932`）
   - 出力は`utf-8-sig`（BOM付きUTF-8）で保存

**実行例**:
```bash
# ドライラン（確認のみ）
python scripts/apply_roman_to_master_csvs.py \
  --roman-dict output/master/player_id_to_roman_full.csv \
  --data-dir _data/master_csv_calculated \
  --dry-run

# 実際に反映
python scripts/apply_roman_to_master_csvs.py \
  --roman-dict output/master/player_id_to_roman_full.csv \
  --data-dir _data/master_csv_calculated
```

**統計情報**:
- 処理したCSVファイル数
- 総更新数（`romanName`が設定された行数）
- 総スキップ数（既に値がある、または`player_id`が辞書にない行数）

## データフロー

```
1. 既存CSVファイル
   ↓
2. all_player_ids.csv（全player_idリスト）
   ↓
3. build_player_name_kana_and_official_roman.py
   ↓
4. player_id_name_kana_official.csv（かな・英字の生データ）
   ↓
5. build_player_id_to_roman_full.py
   ↓
6. player_id_to_roman_full.csv（最終辞書: player_id → romanName）
   ↓
7. apply_roman_to_master_csvs.py
   ↓
8. 成績CSVファイル（player_name_en列に反映）
```

## 技術的な詳細

### ヘボン式ローマ字変換

- **変換テーブル**: `HEPBURN_TABLE`（ひらがな・カタカナ → ローマ字のマッピング）
- **特殊処理**:
  - 長音記号（ー）: 前の文字の母音を長音化（例: `aa` → `ā`）
  - 促音（っ）: 次の子音を重ねる（例: `っか` → `kka`）
  - 拗音: 2文字の組み合わせを優先的にチェック（例: `きゃ` → `kya`）

### 名前の形式変換

- **日本人選手**: 「苗字 名前」→「名前のイニシャル.苗字」
  - 例: "Sato Teruaki" → "T.Sato"
- **外国人選手**: 「名前 苗字」→「名前のイニシャル.苗字」
  - 例: "Roman Mejias" → "R.Mejias"

### HTMLパース戦略

1. **優先1**: `id="pc_v_name"`の`<li>`要素内の括弧内の英字（外国人選手用）
2. **優先2**: `id="pc_v_kana"`の`<li>`要素内の括弧内の英字（日本人選手用）
3. **優先3**: `id="pc_v_kana"`の`<li>`要素内の直接の英字（外国人選手用、括弧なし）

### エラーハンドリング

- **リトライ機能**: ネットワークエラーやタイムアウト時に最大5回リトライ
- **レート制限**: デフォルト1秒間隔でリクエスト（`--rate`オプションで調整可能）
- **HTTP 429**: レート制限エラー時は指数バックオフ（1s, 2s, 4s, 8s, 16s, 最大20s）
- **HTTP 500**: サーバーエラー時も指数バックオフでリトライ
- **キャッシュ機能**: HTMLをローカルにキャッシュし、再実行時に再利用

## 実行時間の目安

- **STEP 3（公式かな・英字の収集）**: 
  - 全`player_id`数 × 1秒（レート制限） = 約数時間〜数日（数千件の場合）
  - キャッシュ機能により、再実行時は大幅に短縮
- **STEP 4（かな→ローマ字変換）**: 数分〜数十分（数千件の場合）
- **STEP 5（成績CSVへの反映）**: 数分（CSVファイル数による）

## 注意事項

1. **NPB公式サイトへの負荷**: レート制限を守り、過度なリクエストを避ける
2. **データの正確性**: `roman_official`が存在する場合はそれを優先（公式データ）
3. **かなの品質**: かなが不完全な場合、ローマ字変換の精度が低下する可能性
4. **文字コード**: Windows環境では`utf-8-sig`（BOM付きUTF-8）を使用
5. **継続実行**: 大量の`player_id`を処理する場合は`--resume`オプションで中断・再開可能

## 2025年データへの適用

2025年データについては、以下の方法で英字名前を追加しました：

1. **スクレイピングスクリプト内でのマッピング**:
   - `scripts/scrape_npb_batting_stats.py`の`get_player_english_name()`関数に手動マッピングを追加
   - 新規選手（ファビアン、西川史礁など）の英字名前を直接マッピング

2. **フルネーム形式での保存**:
   - 外国人選手: "Sandro Fabian", "Jose Reyes", "Tyler Nevin"など
   - 日本人選手: "Nishikawa Shisho", "Makihara Taisei"など

3. **表示時の変換**:
   - `components/RankingUI.tsx`の`formatRomanName()`関数で「イニシャル.苗字」形式に変換
   - 例: "Sandro Fabian" → "S.Fabian", "Nishikawa Shisho" → "S.Nishikawa"

## まとめ

2024年以前の英字名前実装は、以下の5段階のプロセスで完了しました：

1. **player_idリストの準備**
2. **NPB公式サイトの構造調査**
3. **公式かな・英字の収集**（`build_player_name_kana_and_official_roman.py`）
4. **かな→ローマ字変換 & 最終辞書作成**（`build_player_id_to_roman_full.py`）
5. **成績CSVへの反映**（`apply_roman_to_master_csvs.py`）

このプロセスにより、2024年以前の全選手データに対して、NPB公式サイトから取得した英字名前またはかなから変換したローマ字を「イニシャル.苗字」形式で成績CSVに反映することができました。
