# 野手（打撃成績）ランキングページ実装プロセス報告書

## 概要

本報告書は、NPB公式サイトからのスクレイピングからランキングページ完成に至るまでの全プロセスを記録したものです。投手版を作成する際の参考資料として、特に**発生しがちな問題点（ネガティブポイント）**を詳しく記録しています。

---

## 1. データフロー全体図

```
NPB公式サイト
    ↓ [スクレイピング]
生CSV (batting_YYYY_LL_from_master.csv)
    ↓ [指標計算]
計算済みCSV (_data/master_csv_calculated/batting_YYYY_LL_from_master.csv)
    ↓ [読み込み・変換]
RankingViewModel
    ↓ [UI表示]
ランキングページ (/ranking/[year]/[league])
```

##### ❌ 問題8: 列名の正規化が不完全
- **症状**: 全角スペース、BOM、前後空白が混在している

```python
def normalize_column_name(col_name: str) -> str:
    if not col_name:
        return col_name
    # BOM除去
    col_name = col_name.lstrip('\ufeff')
    # 前後空白除去
    col_name = col_name.strip()
    # 全角スペース除去
    col_name = col_name.replace('\u3000', ' ').strip()
    # 連続空白を1個に

    return col_name
```

##### ❌ 問題10: 依存指標の計算順序

```python
def parse_batting_filename(filename: str) -> Optional[Dict[str, Any]]:
    pattern = r'batting_(\d{4})_(PL|CL)_from_master\.csv'
    match = re.match(pattern, filename)
    if match:
        return {'year': int(match.group(1)), 'league': match.group(2)}
    return None
```

### フェーズ4: データ変換（アダプター）

**ファイル**: `lib/ranking/adapter.ts`

#### 処理内容
1. CSV行を`RankingRow`形式に変換
2. 各指標の値を取得（CSVから直接取得 or 計算）
3. 計算可能な指標は計算値を優先（CSVの値が間違っている場合があるため）
4. フォーマット（小数点以下3桁など）

#### 発生しがちな問題点（最重要）

##### ❌ 問題18: CSVの値が間違っている（2025年特有、最重要）
- **症状**: 
  - SLGがAVGと同じ値になっている（例: SLG=0.277, AVG=0.277）
  - OPSが異常に低い値になっている（例: OPS=0.365、実際は0.752）
  - IsoPが負の値になっている（SLG < AVG のため）
- **根本原因**: CSV生成時の計算ロジックに問題がある可能性
- **対策**: **計算値を常に優先する**ロジックを実装
```typescript
// 打率は常に計算値（H/AB）を使用
if (csvKey === '打率' || csvKey === 'AVG' || csvKey === 'avg') {
  const h = getNumericValue(row, ['H', 'h', 'Hits', 'hits', '安打'])
  const ab = getNumericValue(row, ['AB', 'ab', '打数'])
  if (ab !== null && ab > 0 && h !== null && h >= 0) {
    return h / ab  // CSVの値は無視
  }
}

// SLGがAVGと同じ値の場合は計算値を優先
if (csvKey === '長打率' || csvKey === 'SLG') {
  const calculatedSlg = calculateSlg(row)  // TB / AB
  const csvSlg = getNumericValue(row, ['SLG', 'slg', '長打率'])
  const avg = getNumericValue(row, ['AVG', 'avg', '打率'])
  if (csvSlg !== null && avg !== null && Math.abs(csvSlg - avg) < 0.001) {
    return calculatedSlg  // CSVの値がAVGと同じ場合は計算値を返す
  }
}

// IsoPとIsoDは常に計算値を使用
if (csvKey === 'IsoP' || csvKey === 'isop') {
  const slg = calculateSlg(row)  // 計算値から取得
  const avg = calculateAvg(row)   // 計算値から取得
  if (slg !== null && avg !== null) {
    return slg - avg
  }
}
```

##### ❌ 問題22: 単打（1B）の計算
- **症状**: CSVに「1B」列が存在しない場合がある
- **対策**: H - 2B - 3B - HR で計算
```typescript
if (csvKey === '単打' || csvKey === '1B' || csvKey === 'singles') {
  const h = getNumericValue(row, ['H', 'h', 'Hits', 'hits', '安打']) ?? 0
  const doubles = getNumericValue(row, ['2B', 'doubles', '二塁打']) ?? 0
  const triples = getNumericValue(row, ['3B', 'triples', '三塁打']) ?? 0

    }
```

##### ❌ 問題23: 塁打（TB）の計算
- **症状**: CSVに「TB」列が存在しない場合がある
- **対策**: 1B + 2*2B + 3*3B + 4*HR で計算
```typescript
function calculateTB(row: BattingCsvRow): number | null {
  const tb = getNumericValue(row, ['TB', 'tb', '塁打'])
  if (tb !== null) {
    return tb
  }
  // TBが空の場合は計算
  const h = getNumericValue(row, ['H', 'h', 'Hits', 'hits', '安打']) ?? 0
  const doubles = getNumericValue(row, ['2B', 'doubles', '二塁打']) ?? 0
  const triples = getNumericValue(row, ['3B', 'triples', '三塁打']) ?? 0
  const hr = getNumericValue(row, ['HR', 'hr', '本塁打']) ?? 0
  if (h > 0) {
    const singles = h - doubles - triples - hr
    if (singles >= 0) {
      return singles + 2 * doubles + 3 * triples + 4 * hr
    }
  }
  return null
}
```

#### 発生しがちな問題点

##### ❌ 問題31: adapter.tsとleaders.tsのロジック不一致
- **症状**: ランキングページでは正しく表示されるが、トップページのリーダー表示が間違っている
- **原因**: `leaders.ts`の`getMetricValueForLeader`関数が`adapter.ts`の`getMetricValue`関数と異なるロジックを使用している
- **対策**: **同じロジックを使用する**（計算値を優先するロジックを`leaders.ts`にも実装）
```typescript
// leaders.tsでも、adapter.tsと同じロジックを使用
// 打率は常に計算値を使用
if (csvKey === '打率' || csvKey === 'AVG' || csvKey === 'avg') {
  const h = getNumericValue(row, ['H', 'h', 'Hits', 'hits', '安打'])
  const ab = getNumericValue(row, ['AB', 'ab', '打数'])
  if (ab !== null && ab > 0 && h !== null && h >= 0) {
    return h / ab
  }
}
```

## 3. 投手版を作成する際のチェックリスト

### データ構造の違い

1. **指標名の違い**
   - 打撃: OPS, AVG, HR, RBI, ...
   - 投手: ERA, WHIP, W, L, SV, ...

2. **計算式の違い**
   - 打撃: OPS = OBP + SLG, AVG = H / AB
   - 投手: ERA = (ER * 9) / IP, WHIP = (BB + H) / IP

3. **規定到達の基準**
   - 打撃: 規定打席（PA）
   - 投手: 規定投球回（IP）または試合数（G）

### 実装時の注意点

1. **スクレイピングスクリプト**
   - URL: `bat_{league_code}.html` → `pit_{league_code}.html`
   - 列名マッピング: 打撃用 → 投手用に変更

2. **計算スクリプト**
   - `compute_metrics_all_seasons.py`を投手用にコピー
   - 計算式を投手用に変更

3. **loaders.ts**
   - `loadBattingCsv` → `loadPitchingCsv`
   - CSVファイル名: `batting_*` → `pitching_*`

4. **adapter.ts**
   - `getMetricValue`関数を投手用に実装
   - 計算式を投手用に変更（ERA, WHIP, FIPなど）

5. **qualifyingPA.ts**
   - `METRICS_REQUIRE_QUALIFYING_PA` → `METRICS_REQUIRE_QUALIFYING_IP`（または試合数）
   - 規定投球回数の計算ロジックを実装

6. **Record.csv**
   - 投手用の`Record.csv`を作成（指標の順番を定義）

7. **leaders.ts**
   - `getBattingLeaders` → `getPitchingLeaders`
   - トップ3指標: `['OPS', '打率', '本塁打']` → `['防御率', '最多勝', 'セーブ']`など

### 最重要ポイント

1. **CSVの値は信用しない（特に2025年）**
   - 計算可能な指標は常に計算値を優先
   - 打率、SLG、OBP、OPS、IsoP、IsoDは特に注意

2. **列名の表記揺れに対応**
   - 「盗塁死」と「盗塁刺」の両方に対応
