# 2025年英字名前実装計画書

## 概要

2025年の選手データに対して、NPB公式サイトから英字名前（romanName）を収集し、成績CSVファイルに反映する実装計画です。2024年以前の実装プロセスを踏襲しつつ、2025年特有の特殊ケース（外国人選手のかっこ内スペル）に対応します。

## 実装段階

### STEP 1: player_idリストの準備

**目的**: 2025年の全選手の`player_id`を収集し、処理対象を明確化する

**処理内容**:
- 2025年の成績CSVファイル（`_data/master_csv_calculated/batting_2025_CL_from_master.csv`、`batting_2025_PL_from_master.csv`）から`player_id`を抽出
- `player_id`が空の場合は、`player_name_ja`と`team`の組み合わせで識別
- ユニークな`player_id`（または`player_name_ja + team`の組み合わせ）のリストを作成
- 出力: `output/master/all_player_ids_2025.csv`

**必要なスクリプト**: `scripts/build_all_player_ids_2025.py`（新規作成）

**注意事項**:
- 2025年のデータは`player_id`が空の場合があるため、`player_name_ja`と`team`の組み合わせも識別子として使用
- 外国人選手も含めて全選手を対象とする

---

### STEP 2: NPB公式サイトの構造調査（2025年対応）

**目的**: 2025年のNPB公式サイトの選手個人ページから英字名前を取得する方法を確立する

**処理内容**:
- `scripts/probe_player_page_name_fields_2025.py`を使用して、2025年のNPB公式サイトの選手個人ページ（`https://npb.jp/bis/players/{player_id}.html`）のHTML構造を調査
- 以下の要素から英字名前を抽出可能であることを確認:
  - `id="pc_v_name"`の`<li>`要素内の括弧内の英字（外国人選手用）
  - `id="pc_v_kana"`の`<li>`要素内の括弧内の英字（日本人選手用）
  - `id="pc_v_kana"`の`<li>`要素内の直接の英字（外国人選手用、括弧なし）

**特殊ケース: 外国人選手のかっこ内スペル（3件）**

2025年には、外国人選手で以下のような特殊な例が3つ存在します：
- かっこ内にスペルが記載されている
- スペルのままスクレイピングする必要がある

**対応方法**:
- 以前の実装（`build_player_name_kana_and_official_roman.py`）を踏襲
- `find_roman_name()`関数で、かっこ内の英字を優先的に抽出
- パターン: `[（(]([A-Za-z\s\.\-\']+)[）)]` で括弧内の英字を抽出
- 組織名（NIPPON、PROFESSIONAL、BASEBALL、ORGANIZATION）を除外
- Title Caseに変換（大文字小文字を適切に）

**出力**: 調査結果のレポート（`_data/reports/2025_player_page_structure_investigation.md`）

**実行例**:
```bash
python scripts/probe_player_page_name_fields_2025.py \
  --ids output/master/all_player_ids_2025.csv \
  --limit 20 \
  --output _data/reports/2025_player_page_structure_investigation.md
```

---

### STEP 3: 公式かな・英字の収集（2025年対応）

**スクリプト**: `scripts/build_player_name_kana_and_official_roman_2025.py`（新規作成、既存スクリプトをベースに2025年対応）

**目的**: 2025年の全`player_id`（または`player_name_ja + team`の組み合わせ）について、NPB公式サイトから読み仮名（かな）と英字表記（roman_official）を収集する

**処理内容**:

1. **HTML取得**:
   - 各`player_id`に対して、複数のURL候補を生成（既存ロジックを踏襲）
   - `player_id`が空の場合は、`player_name_ja`からNPB公式サイトを検索（必要に応じて）

2. **特殊ケース対応: 外国人選手のかっこ内スペル**:
   - `find_roman_name()`関数で、かっこ内の英字を優先的に抽出
   - スペルのまま保存（Title Caseに変換はするが、元のスペルを保持）
   - 例: `"ブラッド・エルドレッド　(BRAD ELDRED)"` → `"Brad Eldred"`

3. **HTMLキャッシュ**:
   - 取得したHTMLを`output/html_cache/players_2025/`に保存
   - 次回実行時にキャッシュから読み込み可能

4. **データ抽出**:
   - `find_japanese_name()`: 日本語名を抽出
   - `find_kana_name()`: 読み仮名を抽出
   - `find_roman_name()`: 英字表記を抽出（**かっこ内スペルを優先**）

5. **継続実行機能**:
   - `--resume`オプションで既存の結果CSVから処理済みをスキップ

**出力**: `output/master/player_id_name_kana_official_2025.csv`

**実行例**:
```bash
python scripts/build_player_name_kana_and_official_roman_2025.py \
  --ids output/master/all_player_ids_2025.csv \
  --out output/master/player_id_name_kana_official_2025.csv \
  --rate 1.0 \
  --resume \
  --use-cache
```

### STEP 4: かな→ローマ字変換 & 最終辞書作成（2025年対応）

**スクリプト**: `scripts/build_player_id_to_roman_full_2025.py`（新規作成、既存スクリプトをベースに2025年対応）

**目的**: 読み仮名（かな）をヘボン式ローマ字に変換し、最終的な「イニシャル.苗字」形式の辞書を作成する

**処理内容**:

1. **入力データの統合**:
   - `player_id_name_kana_official_2025.csv`を読み込み
   - 同一`player_id`（または`player_name_ja + team`の組み合わせ）が複数行ある場合、統合

2. **英字名前の生成**:
   - **優先1**: `roman_official`が存在する場合
     - 外国人選手のかっこ内スペルは、そのまま使用（Title Caseに変換）
     - 外国人選手として処理: `to_initial_lastname(full_name, is_japanese=False)`
     - 例: "Brad Eldred" → "B.Eldred"（名前のイニシャル.苗字）
   
   - **優先2**: `name_kana`が存在する場合
     - ヘボン式ローマ字変換テーブルを使用してかなをローマ字に変換
     - 日本人選手として処理: `to_initial_lastname(full_name, is_japanese=True)`
     - 例: "さとう・てるあき" → "Sato Teruaki" → "T.Sato"

3. **特殊ケース: 外国人選手のかっこ内スペル**:
   - かっこ内のスペルは、そのまま使用（変換しない）
   - Title Caseに変換（例: "BRAD ELDRED" → "Brad Eldred"）
   - イニシャル.苗字形式に変換（例: "Brad Eldred" → "B.Eldred"）

**出力**: `output/master/player_id_to_roman_full_2025.csv`

**実行例**:
```bash
python scripts/build_player_id_to_roman_full_2025.py \
  --input output/master/player_id_name_kana_official_2025.csv \
  --output output/master/player_id_to_roman_full_2025.csv
```

---

### STEP 5: 成績CSVへの反映（2025年対応）

**スクリプト**: `scripts/apply_roman_to_master_csvs_2025.py`（新規作成、既存スクリプトをベースに2025年対応）

**目的**: 最終辞書（`player_id_to_roman_full_2025.csv`）の`romanName`を2025年の成績CSVファイルに反映する

**処理内容**:

1. **辞書の読み込み**:
   - `output/master/player_id_to_roman_full_2025.csv`を読み込み
   - `player_id`（または`player_name_ja + team`の組み合わせ） → `romanName`のマッピングを作成

2. **成績CSVファイルの検索**:
   - `_data/master_csv/`と`_data/master_csv_calculated/`から
   - `batting_2025_(PL|CL)_from_master.csv`パターンのファイルを検索

3. **CSVファイルへの適用**:
   - 各CSVファイルを読み込み
   - `player_id`列を探す（空の場合は`player_name_ja + team`の組み合わせでマッチング）
   - `player_name_en`列を探す（存在しない場合は追加）
   - 各行について:
     - `player_id`（または`player_name_ja + team`）が辞書に存在する場合、`romanName`を設定
     - 既に「イニシャル.苗字」形式の値がある場合はスキップ

**実行例**:
```bash
# ドライラン（確認のみ）
python scripts/apply_roman_to_master_csvs_2025.py \
  --roman-dict output/master/player_id_to_roman_full_2025.csv \
  --data-dir _data/master_csv_calculated \
  --year 2025 \
  --dry-run

# 実際に反映
python scripts/apply_roman_to_master_csvs_2025.py \
  --roman-dict output/master/player_id_to_roman_full_2025.csv \
  --data-dir _data/master_csv_calculated \
  --year 2025
```

## 技術的な詳細

### ヘボン式ローマ字変換

- **変換テーブル**: `HEPBURN_TABLE`（ひらがな・カタカナ → ローマ字のマッピング）
- **特殊処理**:
  - 長音記号（ー）: 前の文字の母音を長音化
  - 促音（っ）: 次の子音を重ねる
  - 拗音: 2文字の組み合わせを優先的にチェック

### 名前の形式変換

- **日本人選手**: 「苗字 名前」→「名前のイニシャル.苗字」
  - 例: "Sato Teruaki" → "T.Sato"
- **外国人選手**: 「名前 苗字」→「名前のイニシャル.苗字」
  - 例: "Brad Eldred" → "B.Eldred"
  - **特殊ケース**: かっこ内スペルはそのまま使用

### HTMLパース戦略（2025年対応）

1. **優先1**: `id="pc_v_name"`の`<li>`要素内の括弧内の英字（外国人選手用、**かっこ内スペルを優先**）
2. **優先2**: `id="pc_v_kana"`の`<li>`要素内の括弧内の英字（日本人選手用）
3. **優先3**: `id="pc_v_kana"`の`<li>`要素内の直接の英字（外国人選手用、括弧なし）

### エラーハンドリング

- **リトライ機能**: ネットワークエラーやタイムアウト時に最大5回リトライ
- **レート制限**: デフォルト1秒間隔でリクエスト
- **HTTP 429**: レート制限エラー時は指数バックオフ
- **HTTP 500**: サーバーエラー時も指数バックオフでリトライ
- **キャッシュ機能**: HTMLをローカルにキャッシュし、再実行時に再利用

## 実行時間の目安

- **STEP 1（player_idリストの準備）**: 数分
- **STEP 2（NPB公式サイトの構造調査）**: 数十分（サンプル調査の場合）
- **STEP 3（公式かな・英字の収集）**: 
  - 全`player_id`数 × 1秒（レート制限） = 約数時間（数十件の場合）
  - キャッシュ機能により、再実行時は大幅に短縮
- **STEP 4（かな→ローマ字変換）**: 数分
- **STEP 5（成績CSVへの反映）**: 数分

---

## 注意事項

1. **NPB公式サイトへの負荷**: レート制限を守り、過度なリクエストを避ける
2. **データの正確性**: `roman_official`が存在する場合はそれを優先（公式データ）
3. **特殊ケース: かっこ内スペル**: 3件の外国人選手のかっこ内スペルは、そのまま使用
4. **player_idが空の場合**: `player_name_ja + team`の組み合わせで識別
5. **文字コード**: Windows環境では`utf-8-sig`（BOM付きUTF-8）を使用
6. **継続実行**: 大量の`player_id`を処理する場合は`--resume`オプションで中断・再開可能
